Principios de la poo (Programación Orientada a objetos)

La programación Orientada a objetos se define como un paradigma de la programación, una manera de programar específica, donde se organiza el código en unidades denominadas clases, de las cuales se crean objetos que se relacionan entre sí para conseguir los objetivos de las aplicaciones. La programación Orientada a objetos (POO) es una forma especial de programar, más cercana a como expresaríamos las cosas en la vida real que otros tipos de programación. Para q un lenguaje de programacion sea considerado como orientado a objetos debe tener dos requisitos:

Primer requisito, un lenguaje debe permitir que un desarrollador describa el realidad utilizando objetos y para definir relaciones entre objetos como los siguientes: Asociación: esta es la capacidad del objeto para referir otro objeto independiente Agregación: esta es la capacidad del objeto para incrustar uno o más independientes objetos Composición: esta es la capacidad del objeto para incrustar uno o más dependientes objetos.

El segundo requsitos son los pilares fundamentales de la poo que son: Encapsulamiento: Es la capacidad de ocultar los datos abstraídos, aislarlos o protegerlos de quién no desees que tenga acceso a ellos; otro objeto o función por ejemplo.

Herencia: este es el mecanismo por el cual un objeto adquiere algunos o todas características de uno o más objetos.

Polimorfismo: esta es la capacidad de procesar objetos de manera diferente en función de su tipo o estructura de datos.

¿Es orientado a objetos JavaScript?

Muchos desarrolladores no consideran a JavaScript como un verdadero lenguaje orientado a objetos debido a su falta de concepto de clase y porque no exige el cumplimiento de los principios de OOP. JavaScript no hace referencia a las clases directamente pero se requieren características y principios para los objetos. Las clases no son un requisito real, pero a veces son una forma conveniente de abstraer conjuntos de objetos con propiedades comunes. Además, los principios OOP requeridos para un idioma están destinados a ser compatibles. Ellos no debe ser obligatorio para poder programar en un lenguaje. El desarrollador puede elige usar construcciones que le permitan crear código orientado a objetos o no.

Abstracción

La abstracción es el concepto de mover el foco de la detalles e implementación concreta de cosas a las características que son relevantes para un determinado propósito, con un enfoque más general y abstracto. En otras palabras, la abstracción es la capacidad para definir qué propiedades y acciones de una entidad del mundo real tienen que ser representado por medio de objetos en un programa para resolver un problema específico. Gracias a ello, podemos representar las características esenciales de un objeto sin preocuparnos de las restantes.

Asociación

La asociación es una relación entre dos o más objetos donde cada objeto es independiente el uno del otro. Esto significa que un objeto puede existir sin el otro y ningún objeto posee el otro.

Agregación

La agregación es el mecanismo que le permite crear un objeto que consta de varios objetos, mientras que la asociación relaciona objetos autónomos.

Composición

La composición es un tipo fuerte de agregación, donde cada objeto componente no tiene vida independiente sin su dueño, el agregado.

Principios de la programación orientada a objetos

El segundo requisito que nos permite considerar JavaScript como un objeto orientado el lenguaje implica el apoyo de al menos tres principios: encapsulación, herencia y polimorfismo. Analicemos cómo JavaScript admite cada uno de estos principios

Encapsulación

Es la característica de la POO que permite el ocultamiento de la complejidad del código, pertenece a la parte privada de la clase y que no puede ser vista desde ningún otro programa.

Herencia

Es el pilar más fuerte que asegura la reutilización de código, ya que a partir de esta característica es posible reutilizar (heredar) las características y comportamientos de una clase superior llamada clase padre, a sus clases hijas, denominadas clases derivadas.

Polimorfismo

A través de esta característica es posible definir varios métodos o comportamientos de un objeto bajo un mismo nombre, de forma tal que es posible modificar los parámetros del método, o reescribir su funcionamiento, o incrementar más funcionalidades a un método.

Los objetos JavaScript no se preocupan por la privacidad. Todas las propiedades y métodos son públicos. accesible si no se toma precaución. Entonces, si queremos evitar el acceso a algunas propiedades o métodos relacionados con los detalles de implementación interna, tenemos que establecer una estrategia.

Un primer enfoque simple consiste en adoptar nombres basados ​​en convenciones para miembros de un objeto. Por ejemplo, los miembros internos pueden tener un nombre que comience con un prefijo, como el carácter de subrayado (_). Vamos a explicar con un ejemplo:

function asientosTeatro() {

this.asientos = [];

}

asientosTeatro.prototype.placePerson = function(person) {

this._asientos.push(person);

};

Niveles de privacidad con cierre:

Una manera simple de corregir las inconsistencias del enfoque basado en convenciones es no usar propiedades para miembros internos pero declarando variables dentro del constructor, como se muestra en El siguiente ejemplo:

function TheatreSeats() {

var seats = [];

this.placePerson = function(person) {

seats.push(person);

};

}
